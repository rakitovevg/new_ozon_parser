# Передача проекта заказчику

Краткий чек-лист: что подготовить и что отдать.

---

## Перед передачей

- [ ] **Удалить свои секреты из репозитория**  
  Убедитесь, что в коде и в истории git нет реальных токенов, паролей, ключей. Файл `.env` не должен коммититься (он в `.gitignore`). Если случайно коммитили — смените все ключи (Telegram-бот, серверы и т.д.).

- [ ] **Проверить .env.example**  
  В нём только названия переменных и пустые/примерные значения, без реальных токенов и паролей.

- [ ] **Обновить README**  
  В README должно быть понятное описание проекта, как установить, как запустить, что настроить в `.env`. Актуальные возможности и ссылка на деплой (DEPLOY.md), если отдаёте с деплоем.

- [ ] **Прогнать проект локально**  
  Создать задачу, запустить, проверить уведомление в Telegram и что админка открывается.

## Что отдать заказчику

### 1. Репозиторий (архив или доступ)

- **Вариант А:** архив (ZIP) с проектом **без** папок `.venv`, `__pycache__`, `data/*.db`, без файла `.env`.
- **Вариант Б:** доступ к репозиторию (GitHub/GitLab и т.п.) — заказчик клонирует сам. В репо не должно быть `.env`, только `.env.example`.

### 2. Документация (уже в репо)

| Файл | Для кого |
|------|----------|
| **README.md** | Описание проекта, установка, запуск, настройка `.env`. |
| **DEPLOY.md** | Пошаговый деплой на сервер (Docker, GitHub Actions, секреты). |
| **HANDOVER.md** | Этот чек-лист (можно не отдавать заказчику, только себе). |

### 3. Письмо/инструкция заказчику (коротко)

Напишите заказчику отдельным письмом или в README:

- Что делает проект (1–2 предложения).
- Как запустить локально: `python -m venv .venv`, `pip install -r requirements.txt`, скопировать `.env.example` в `.env`, заполнить Telegram и URL, запустить `uvicorn app.main:app --port 8000`.
- Где админка: `http://localhost:8000` (или домен после деплоя).
- Что обязательно заполнить в `.env`: `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`, `SEARCH_URL1`, `SEARCH_URL2`.
- Если передаёте с деплоем: ссылка на DEPLOY.md и что нужно создать бота в Telegram, при необходимости — сервер и секреты в GitHub.

### 4. Данные и доступы (по договорённости)

- Если заказчик хочет деплой на свой сервер — передать только инструкцию (DEPLOY.md) и, при необходимости, помочь один раз с настройкой секретов.
- Токены и пароли заказчик создаёт сам (Telegram-бот, GitHub, сервер) и подставляет в `.env` и в секреты GitHub. Реальные значения вам передавать не обязательно.
